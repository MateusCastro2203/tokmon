INSTALAÇÃO
1

npm install -g tokmon-app — instala o comando tokmon globalmente.

2

tokmon setup — integra com o Claude Code (skill, comando /tokens, statusline). Opcional, mas recomendado.

3

Pronto. Abra uma sessão nova do Claude Code pra ver a statusline, ou rode tokmon watch numa aba separada.

COMANDOS
tokmon snapshot [cwd]

Imprime um snapshot de texto da sessão ativa: total de tokens, ranking dos prompts mais caros, quebra por skill/agente, e % de context window. Sem argumento, usa o diretório atual.

tokmon watch [cwd]

Dashboard ao vivo no terminal — atualiza sozinho a cada nova mensagem. Mostra o mascote em pixel art, barra de contexto colorida (verde/amarelo/vermelho), ranking de prompts e quebra por skill. Ctrl+C sai.

tokmon setup

Configura a integração com o Claude Code: symlinka a skill em ~/.claude/skills/token-monitor, copia o comando /tokens, e registra a statusLine em ~/.claude/settings.json — sem apagar o resto do arquivo, e sem sobrescrever uma statusline diferente que você já tenha.

Flags: --force — sobrescreve uma statusline existente
tokmon statusline

Modo interno — lê o JSON que o próprio Claude Code manda pelo stdin e imprime a linha compacta do rodapé. Você não roda isso manualmente; é o que fica registrado no settings.json depois do tokmon setup.

tokmon help

Mostra a lista de comandos no terminal.

INTEGRAÇÃO COM O CLAUDE CODE

Depois do tokmon setup, duas coisas ficam disponíveis dentro do próprio Claude Code:

  • Comando /tokens — pergunta pro Claude "quanto gastei de token" e ele já sabe rodar o snapshot.
  • Statusline nativa — uma linha sempre visível no rodapé do terminal, sem precisar rodar nada, atualizando sozinha a cada resposta.
PRIVACIDADE
100% local. Zero chamada de rede, zero telemetria, zero chave de API. A única coisa lida é o .jsonl que o próprio Claude Code já grava no seu disco — nada sai da sua máquina. (Até esta página: a fonte pixelada está embutida no HTML, sem carregar nada de fora.)
